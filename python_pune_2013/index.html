Challenges

Performance and Language Speed.

Is it really matters ?

No. Python has enough speed to meet Most of the uses cases.

But if we need better performance, what we will do ?

  • Know more about language, and use standard libraries available to improve the performance. eg; itertools, operator, etc.
  • Use Multi Processor package. The Threading module is not much useful.
  • Based on the application ( Web based or others ), pick good set of tools. eg; Database, Caching system, etc.

But if we need better performance, what we will do ?

  • Asynchronous libraries - Twisted, Tornado, Gevent, etc.
  • Pypy -- A good step towards JIT compiled Python Bytecode
  • Use Python C-APIs and write code in C and use it with python
  • There are several tools available to write code in Python and generate its C code. eg; Cython.

Python-3.x is not 100% compatible with Python-2.x

  • Small Changes in the Language syntax.
  • eg; print is a statement in python-2.x, but in python-3.x it's a normal build-in function. 
    •  on Python-2.x print "Hello World"  
     on Python-3.x  print("Hello World")

Python-3.x is not 100% compatible with Python-2.x

  • Lot of memory optimization for different operations like, file read, dict - keys(), values() functions etc. are by default returns an iterator
  • No need to worry, obviously the changes and optimizations makes python-3.x better than python-2.x.
  • Python-2.7.x in cooperate lot of python3 features to make the migration easy. So the beginners can start from python-2.7.x.
